1. Guia de Introdução: A Tríade OpenCRO

Sua porta de entrada para o ecossistema profissional gratuito de IA para todos.

O que é o ecossistema OpenCRO?

O OpenCRO não é uma ferramenta única, mas sim um ecossistema integrado que une três pilares essenciais de código aberto para criar o ambiente ideal de desenvolvimento assistido por IA no Windows:

  • C (Code - OpenCode): O agente de IA local e o aplicativo visual amigável que interage com seu projeto, lê seus arquivos e executa tarefas automaticamente.
  • R (Router - OpenRouter): O gateway unificado que conecta o OpenCode a dezenas de modelos mundiais (Claude, GPT, Gemini, DeepSeek) sob demanda, pagando apenas pelo que usar (centavos por milhão de tokens) ou usando opções 100% gratuitas.
  • O (Okam - Okam CLI): O validador de governança e segurança local que impede vazamento de senhas/chaves API e executa ganchos automáticos (hooks) de revisão de código e skills.

O que é OpenCode?

OpenCode é o seu assistente de IA. Ele possui tanto uma interface gráfica visual (App Desktop) super amigável para iniciantes quanto uma interface de terminal (CLI/TUI) para desenvolvedores. Diferente dos chats comuns (como ChatGPT), o OpenCode lê o contexto de arquivos locais e ajuda você a programar de verdade, sem complicação.

💡
Noob-Friendly: Se você tem medo de terminal, não se preocupe! O App Desktop do OpenCode permite que você abra pastas do Windows e converse com a IA visualmente por meio de janelas comuns.

2. Configuração & Setup no Windows

Escolha o caminho ideal para você: a interface gráfica simples do App Desktop ou a velocidade do terminal.

Passo 1: Criar conta grátis no OpenRouter

Antes de tudo, você precisa da chave de conexão que liga seu assistente aos modelos de linguagem:

  1. Acesse o site oficial do OpenRouter.ai ↗ e cadastre-se gratuitamente (via Google ou GitHub).
  2. Vá para openrouter.ai/keys ↗ e clique em Create Key.
  3. Dê um nome para sua chave (ex: "OpenCode App") e clique em criar.
  4. Copie a chave gerada imediatamente (ela começa com sk-or-...) e guarde em local seguro. Ela só é exibida uma única vez.
1. Baixar o OpenCode App

O OpenCode possui um aplicativo completo para Windows com janelas e interface amigável:

  1. Acesse o site oficial opencode.ai ↗.
  2. Clique no botão para baixar o instalador de Windows (arquivo .exe).
  3. Execute o instalador baixado e siga as instruções na tela.
2. Configurar Chave de API no App

Ao abrir o aplicativo pela primeira vez:

  1. Clique no ícone de Engrenagem (Configurações) no canto inferior esquerdo.
  2. Selecione OpenRouter como o provedor de IA.
  3. Cole a sua chave de API (sk-or-...) no campo correspondente e salve.
  4. No menu suspenso de modelos, selecione seu modelo inicial. Recomendamos o deepseek/deepseek-v4-flash (super barato) ou o hy3 (gratuito).
3. Abrir seu Projeto

Para interagir com seus arquivos: clique em Open Folder (Abrir Pasta) e selecione o diretório dos seus arquivos no Windows. O OpenCode carregará todo o contexto visualmente e você poderá começar a solicitar alterações escrevendo na caixa de bate-papo.

1. Instalação via WSL (Linux recomendado no Windows)

Se você prefere a linha de comando e TUI no terminal, execute no PowerShell como Administrador:

wsl --install

Após reiniciar e configurar o Ubuntu, instale o OpenCode dentro dele:

curl -fsSL https://opencode.ai/install | bash
source ~/.bashrc
2. Instalação Nativa via npm (Node.js)

Se não quiser WSL, instale no Windows pelo Node.js:

npm install -g opencode-ai
3. Configurar Chave no Terminal

Configure a variável no seu terminal para conectar ao OpenRouter:

[System.Environment]::SetEnvironmentVariable('OPENROUTER_API_KEY', 'sk-or-v1-sua-chave-aqui', 'User')

Passo 2: Governança do Projeto (AGENTS.md)

No App Desktop ou no terminal, ao abrir um projeto novo pela primeira vez, o assistente perguntará se deseja inicializar as regras de governança. Alternativamente, você pode rodar o comando /init.

Isso criará o arquivo AGENTS.md na raiz da sua pasta. Nele, você pode escrever regras simples em português (ex: "Não use banco de dados MySQL, prefira SQLite", "mantenha os arquivos pequenos e explicados"). A IA lerá essas instruções antes de qualquer alteração, garantindo total controle.

Otimizando seus Gastos no OpenRouter

Se você optar por usar modelos pagos além dos modelos free, use a estratégia de alternância inteligente para economizar:

  • Use os modelos gratuitos hy3 ou nemotron-3-ultra para tarefas repetitivas, perguntas de documentação e revisão simples.
  • Use o modelo deepseek/deepseek-v4-flash ou mimo-v2.5 para desenvolvimento ativo devido ao custo de entrada extremamente baixo (~$0.09 a $0.10 por milhão de tokens).
  • Reserve o deepseek/deepseek-v4-pro, claude/claude-opus-4.8 ou gpt-5.5 para tarefas de grande complexidade arquitetural, refatorações multi-arquivo ou debug profundo.
⚠️
Dica de Economia: Use frequentemente o comando /clear para esvaziar a conversa quando concluir uma subtarefa. Sessões muito longas acumulam contexto, o que aumenta o consumo de tokens a cada pergunta.

O que é o Okam CLI?

O Okam é uma ferramenta complementar e open-source para a governança local de agentes de IA. Ele atua como um gerenciador de diretrizes ("caderno de regras e memórias"), garantindo que os assistentes de IA respeitem as convenções do seu projeto e executem ações automáticas de segurança, como evitar commits com segredos expostos.

1. Instalação Rápida

Instale o Okam globalmente via npm no terminal (PowerShell ou WSL). Requer Node.js >= 18:

npm install -g okam-cli

Verifique a versão para certificar a correta instalação:

okam --version

2. Inicializando no seu Projeto

Navegue até a raiz do seu código e execute o setup inicial:

okam init

Isso gerará o arquivo local de configuração .okamrc.

3. Automação de Governança com Hooks (Ganchos)

Os hooks do Okam permitem disparar comandos no terminal automaticamente antes (pre-command) e depois (post-command) de cada interação com a IA:

hooks:
  pre-command:
    # Atualiza as ramificações locais antes de propor modificações
    - "git fetch origin"
  post-command:
    # Roda o validador de sintaxe ou testes automaticamente para checar a IA
    - "npm run lint && npm test"

4. Auditoria de Código com Pipes do Git

Você pode encadear comandos no terminal para submeter trechos ou o diff completo para revisão da IA antes de fechar um commit:

git diff --cached | okam run "revise este diff e aponte problemas de segurança, credenciais ou segredos expostos"

5. Estendendo com Skills

Skills são diretórios contendo pacotes estruturados de instruções (como regras de Clean Code ou Checklists TDAH). Instale-as facilmente:

# Instalar uma skill a partir do repositório
okam skill install checklists

# Listar skills atualmente ativas na sessão
okam skill list
