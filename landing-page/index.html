1. Guia de Introdução: OpenCode + OpenRouter

Seu primeiro passo para dominar agentes de IA no terminal do Windows.

O que é OpenCode?

OpenCode é um agente de IA open-source que opera diretamente no seu terminal (ou como app desktop/IDE). Ele entende seu projeto, acessa seus arquivos, roda comandos e escreve código — tudo orquestrado por modelos de linguagem (LLMs).

Diferente de chatbots comuns como ChatGPT, o OpenCode tem contexto completo do seu projeto: ele lê a estrutura de diretórios, entende suas dependências, conhece suas convenções de código (via AGENTS.md) e executa ações reais no sistema, como instalar pacotes, criar arquivos e fazer commits.

💡
Características principais: 100% open-source, multi-provedor (funciona com Claude, GPT, Gemini, Llama), multi-sessão (agentes em paralelo), LSP-aware (entende a estrutura do código) e expansível com Skills e MCP.

O que é OpenRouter?

O OpenRouter é um gateway unificado de APIs para modelos de IA. Em vez de criar contas e cadastrar cartões de crédito em múltiplos provedores (OpenAI, Anthropic, Google, Meta...), você utiliza uma única API key para acessar centenas de modelos pagando apenas pelo consumo real de tokens (ou usando modelos gratuitos).

Por que usar juntos?

O OpenCode precisa de modelos de IA para rodar. Ao conectá-lo ao OpenRouter, você ganha acesso a todo o catálogo mundial de IA com uma única chave e uma única conta. É a forma mais econômica e prática de rodar o seu assistente local.

2. Instalação: OpenCode + OpenRouter no Windows

Configure seu ambiente de desenvolvimento assistido por IA em 15 minutos.

⭐
Duas opções de instalação: A Rota A (WSL - Windows Subsystem for Linux) é altamente recomendada devido ao desempenho superior de arquivos e compatibilidade total. A Rota B é nativa no Windows usando Node.js/npm.

Passo 1: Criar conta grátis no OpenRouter

Antes de instalar as ferramentas, você precisa da chave de API que conecta o agente aos modelos de linguagem:

  1. Acesse o site oficial do OpenRouter.ai ↗ e crie uma conta gratuita (pode usar login direto com Google ou GitHub).
  2. Vá para a seção de chaves em openrouter.ai/keys ↗.
  3. Clique em Create Key, dê um nome para ela (exemplo: "opencode-local") e salve.
  4. Importante: Copie e guarde a chave gerada (ela começa com sk-or-...). Ela é exibida apenas uma vez.
1. Instalar WSL (Ubuntu)

Abra o PowerShell como Administrador e execute o comando abaixo para ativar o Linux integrado ao Windows:

wsl --install

Reinicie o computador quando solicitado. Ao reiniciar, o terminal do Ubuntu abrirá automaticamente para você definir seu usuário e senha do Linux.

2. Instalar OpenCode no Ubuntu

No terminal do Ubuntu (WSL), execute o instalador oficial:

curl -fsSL https://opencode.ai/install | bash

Recarregue as configurações de terminal:

source ~/.bashrc
3. Configurar a Chave do OpenRouter no WSL

Crie o diretório de configurações do OpenCode:

mkdir -p ~/.config/opencode

Agora crie o arquivo config.json para salvar sua chave. Você pode colar a estrutura JSON abaixo substituindo sua-chave-aqui:

{
  "provider": "openrouter",
  "openRouterApiKey": "sk-or-v1-sua-chave-aqui",
  "model": "meta-llama/llama-3.1-70b"
}
💡
Modelo Gratuito: Por padrão, definimos o meta-llama/llama-3.1-70b que é 100% gratuito no OpenRouter. Você pode rodar à vontade sem adicionar saldo!
4. Acessando seus projetos do Windows no WSL

Seus arquivos do Windows (C:, D:, etc.) ficam mapeados dentro do WSL na pasta /mnt/. Navegue até seu projeto:

cd /mnt/c/Users/SeuNome/Documents/meu-projeto
opencode
1. Instalar Node.js

Baixe e instale a versão LTS recomendada do Node.js ↗. Certifique-se de marcar a opção para adicionar ao PATH do sistema durante a instalação.

2. Instalar OpenCode via npm

Abra o PowerShell e rode o instalador global:

npm install -g opencode-ai
3. Salvar Variável de Ambiente no Windows

Execute no PowerShell para definir permanentemente sua chave do OpenRouter:

[System.Environment]::SetEnvironmentVariable('OPENROUTER_API_KEY', 'sk-or-v1-sua-chave-aqui', 'User')

Reinicie o terminal do PowerShell após este comando para que a chave seja carregada.

4. Inicializar o OpenCode

Navegue até a pasta do seu projeto e chame o assistente:

cd C:\projetos\meu-projeto
opencode

Passo 2: Configurando seu Projeto (/init)

Na primeira vez que rodar o OpenCode dentro de uma pasta de projeto, envie o comando especial:

/init

Este comando criará o arquivo AGENTS.md na raiz do projeto. Ele funciona como a governança do seu agente, permitindo ditar regras personalizadas de desenvolvimento (exemplo: padrão de código, banco de dados preferido, pastas a evitar).

3. Guia de Uso Avançado & Boas Práticas

Comandos Essenciais do Terminal OpenCode

Comando Ação no Terminal
/help Lista todos os comandos integrados da CLI
/undo Desfaz a última alteração de arquivo gerada pela IA

Otimizando seus Gastos no OpenRouter

Se você optar por usar modelos pagos além dos modelos free, use a estratégia de alternância inteligente para economizar:

  • Use o modelo gratuito meta-llama/llama-3.1-70b para tarefas repetitivas, perguntas de documentação e revisão de código simples.
  • Use o modelo deepseek/deepseek-chat (DeepSeek V3) para desenvolvimento ativo, pois tem um custo de entrada extremamente baixo (~$0.14 por milhão de tokens) e ótima capacidade lógica.
  • Reserve o anthropic/claude-sonnet-4 para tarefas de grande complexidade arquitetural, refatorações multi-arquivo ou debug complexo.
⚠️
Dica de Economia: Use frequentemente o comando /clear para esvaziar a conversa quando concluir uma subtarefa. Sessões muito longas acumulam contexto, o que aumenta o consumo de tokens a cada pergunta.

Bônus: Governança de IA Local com Okam CLI

O Okam é uma ferramenta complementar open-source para governança local de IA. Ele atua como o "caderno de memórias e regras" do seu assistente de IA, gerenciando suas sessões, aplicando ganchos (hooks) de validação e estendendo o assistente com skills especializadas.

1. Instalação Rápida

Instale globalmente via npm no terminal (PowerShell ou WSL) se já tiver o Node.js instalado:

npm install -g okam-cli
2. Inicializando no Projeto

Navegue até a raiz do seu projeto e execute:

okam init

Isso criará o arquivo de configuração local .okamrc.

3. Automação com Hooks (Ganchos)

Adicione validações automáticas antes ou depois de cada comando no seu arquivo .okamrc:

hooks:
  pre-command:
    - "git fetch origin"
  post-command:
    - "npm run lint"
4. Auditoria Pré-Commit (Modo Pipe)

Valide suas alterações de código enviando o diff do Git diretamente para o Okam antes de commitar:

git diff --cached | okam run "revise este diff e aponte falhas de segurança ou credenciais expostas"
5. Gerenciando Skills

Skills estendem as instruções padrão do assistente. Gerencie-as com facilidade:

# Instalar a skill de checklists ADHD-friendly
okam skill install checklists

# Listar skills instaladas
okam skill list

Criação de API REST Completa

Gera um servidor backend completo com validação de dados, banco e tratamento de erros.

Crie uma API REST em [Node.js/Python/Go] com os seguintes endpoints:
- GET /api/items (listar todos)
- POST /api/items (criar um item)
- DELETE /api/items/:id (deletar item)

Use [SQLite/PostgreSQL] como banco. Inclua:
- Validação de dados de entrada
- Tratamento de erros estruturado
- Um README.md detalhando como rodar a aplicação
Script de Automação de Arquivos (Python)

Por que OpenRouter

Um acesso, todos os modelos

Conecte-se a Claude, GPT, Gemini, Llama e centenas de outros modelos através de uma única API. Sem assinatura de cada serviço separadamente.
